# 14. 사전학습 임베딩 + 경량 분류기 프롬프트 (가성비 최강 카드)
목적: fine-tune 없이 frozen 임베딩 추출 -> LGBM/MLP. 30분~1시간 내 CNN급 점수 가능.
fine-tune(08번)이 도는 동안 병렬로 확보하는 보험이자 앙상블 재료.

===== 프롬프트 시작 =====

너는 오디오 표현학습 전문가다. 사전학습 오디오 모델을 frozen 상태로 임베딩 추출기로만 사용하고, 그 위에 경량 분류기를 학습하는 파이프라인을 작성하라. 환경: H200, 인터넷 가능.

[입력]
- folds.csv, 클래스/평가지표: {01·03번 결과}
- 세그먼트: {N}초, 16kHz (모델 호환)

[요구사항]
1. 임베딩 추출기 3종 (공통 인터페이스 extract(wav_batch) -> np.ndarray):
   a. PANNs CNN14 (AudioSet, 2048-d) — 환경음 계열에 강함
   b. AST (CLS 토큰 + 패치 평균 concat)
   c. WavLM-large (전 레이어 hidden states의 학습 가능 가중합 대신,
      frozen이므로 마지막 4개 레이어 평균 풀링)
   GPU 배치 추론, bf16, 전 데이터 임베딩을 emb_{model}.npy로 캐싱
2. 분류기 2종 (임베딩별 x 분류기별 OOF 생성):
   a. LightGBM (05번 규격)
   b. MLP (512-256, dropout 0.3, 50 epochs, early stopping) — 확률 보정 양호
3. 점수 표 출력: 3 임베딩 x 2 분류기 = 6조합의 녹음 단위 OOF 점수 비교
   + 6조합 확률 평균 앙상블 점수
4. 임베딩 concat 실험: 상위 2개 임베딩 concat -> LGBM (차원 증가 대비 효과 확인)
5. 산출물 규격 통일: oof_emb_{name}.npy / test_emb_{name}.npy / 제출 파일 / 실험 로그
6. 보너스: 임베딩 UMAP 2D 시각화 (클래스 색상) — 클래스 분리도 육안 확인 +
   분리 안 되는 클래스 쌍 보고 (이후 오류 분석 연결)

[왜 이 접근인가 — 주석에 포함]
- 추출 1회 후 분류기 실험이 분 단위로 회전 -> 하이퍼파라미터/피처 실험 속도 극대화
- fine-tune 실패/시간 부족 시의 안전망
- 04번 신호처리 피처와 결합(concat) 시 도메인 지식 + 표현학습 결합 효과

===== 프롬프트 끝 =====

## 체크리스트
- [ ] 임베딩 캐싱 후 분류기 재실험이 5분 내 도는가
- [ ] 04번 수치 피처와 concat한 LGBM도 시도했는가
- [ ] UMAP에서 분리 안 되는 클래스 쌍을 오류 분석(17번)으로 넘겼는가
