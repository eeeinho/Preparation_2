# 11. 팀 운영 — 실험관리 시트 + 역할 분담 + 시간 예산
목적: 28.5시간 무박2일에서 혼선/중복작업/기록누락 방지. 대회 시작 전 팀 전체 공유.

## A. 실험 관리 시트 (Google Sheets 또는 CSV)
컬럼 정의 — 보조 팀원 1명이 전담 기록:

exp_id | 담당 | 모델 | 입력특징 | CV방식 | fold점수(5개) | CV평균(녹음단위) | public점수 | 학습시간 | seed | augmentation | 제출파일명 | 메모

운영 규칙:
- 제출 즉시 public 점수 기입, CV vs public 산점도 1시간마다 갱신
- "메모"에는 직전 실험 대비 바꾼 것 단 하나만 기록 (변인 통제)
- 점수 하락 실험도 반드시 기록 (같은 실수 반복 방지)

## B. 역할 분담 (4인: 코딩 2 + 보조 2)
- InhoLee (전략 총괄): EDA 해석, CV 설계 확정, 모델 선택 결정, 앙상블, 최종 제출 판단
- 코딩 팀원: 모델 학습 실행/모니터링, 피처 추출 스크립트 실행, GPU 실험 큐 관리
- 보조1 (운영): 데이터 구조/공지사항/규정 숙지, 리더보드·제출 이력 기록, 실험 시트 관리,
  제출 파일 검증기 실행 담당
- 보조2 (리서치+문서): 논문/유사대회 솔루션 검색, LLM 프롬프트 실행(이 세트 활용),
  오분류 샘플 청취 검수, 발표자료/보고서 초안 작성

핸드오프 규칙:
- 모든 산출물(폴드, OOF, 체크포인트)은 공용 경로 + 이름 규칙 준수 (06/08번 프롬프트 규격)
- 코딩 2명은 같은 folds.csv만 사용 — 변경 시 전원 공지
- 수면 교대: 코딩 2명은 동시에 자지 않음 (학습 모니터링 공백 방지)

## C. 시간 예산 (28.5h) — 마스터 프롬프트와 동일 기준
| 구간 | 작업 | 게이트(통과 조건) |
|------|------|------------------|
| 0~1.5h | 01 구조점검 + 02 EDA + 03 CV 확정 | folds.csv 확정, 누수 위험 목록 작성 |
| 1.5~3h | 04 피처 + 05 LGBM 첫 제출 | 07 검증기 통과, CV-public 1차 확인 |
| 3~9h | 06 CNN + 08 AST 병렬 학습 | fold0 게이트: 베이스라인 대비 향상 확인 |
| 9~12h | OOF 분석, 오분류 청취, 증강 튜닝 | confusion matrix 기반 개선점 1개 이상 |
| 12~20h | 09-A 듀얼브랜치 / 필요시 도메인 적응 | ablation 표로 채택 여부 결정 |
| 20~24h | 09-B 앙상블 + 집계/threshold 튜닝 | 안전/공격 제출 2개 확보 |
| 24~27h | 최종 재학습 판단 + 제출물 패키징 | inference.py 단독 재현 성공 |
| 27~28.5h | 코드/가중치/문서 검수 | 12번 보고서 완성 |

원칙: 항상 제출 가능한 최선의 결과물 유지. 마지막 2시간 신규 실험 금지.
