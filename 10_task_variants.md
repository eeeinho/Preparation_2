# 10. 태스크 변형 대응 프롬프트 4종
목적: 함종 분류가 아닌 주제가 출제될 경우 즉시 전환. 각 프롬프트는 독립 사용 가능.

# ============================================
# A. 함종/위협 수준 분류 (기본 — 출제 확률 최고)
# ============================================
===== 프롬프트 시작 =====
너는 Kaggle Grandmaster + 수중 음향 전문가다. WAV(+레이블 CSV, train/test/sample_submission)가 주어진 함종(또는 위협 수준) 분류 대회다.
순서: (1) 01번 구조점검 -> (2) 03번 그룹 CV 확정 -> (3) 04+05번 LGBM 첫 제출
-> (4) 06번 Mel CNN -> (5) 08번 AST -> (6) 09번 앙상블.
각 단계 해당 프롬프트를 사용하되, 위협 수준이 순서형(ordinal)이면:
- CrossEntropy 외에 회귀 헤드(+반올림) / CORAL ordinal loss 비교 실험 추가
- 평가지표가 가중 페널티(인접 오류 관대)면 QWK 최적화 검토
===== 프롬프트 끝 =====

# ============================================
# B. 이상탐지 (정상 데이터만 또는 극단 불균형)
# ============================================
===== 프롬프트 시작 =====
너는 이상탐지 전문 Kaggle Grandmaster다. 수중 음향 이상탐지 태스크다(레이블: 정상만 존재 또는 이상 비율 < 5%). 환경: H200 GPU, 인터넷 가능.

[입력] {데이터 구조/평가지표 — 01번 결과}

[요구사항]
1. 진단 먼저: 이상 레이블이 일부라도 있는가?
   - 있으면(소수): 지도 학습(LGBM + class weight + PR-AUC 최적화)이 비지도보다 거의 항상 강함
     -> 05번 프롬프트를 PR-AUC/F1 기준으로 변형 사용 + 아래 비지도 스코어를 피처로 추가
   - 전혀 없으면: 순수 비지도 파이프라인
2. 비지도 3종 구현 (04번 피처 기반 + 스펙트로그램 기반):
   a. IsolationForest / One-Class SVM (수치 피처)
   b. Conv AutoEncoder (log-Mel 재구성, anomaly score = 재구성 오차 맵의 상위 분위수 평균)
   c. kNN distance (사전학습 오디오 임베딩 — AST/PANNs 임베딩 추출 후 정상 train과의 거리)
3. 스코어 정규화(rank 또는 min-max) 후 3종 평균 앙상블
4. threshold: 검증 이상 샘플이 있으면 OOF F1 최적화 / 없으면
   train 스코어 분포 분위수(예: 99%) + 제출 2개(보수/공격) 전략
5. CV: 비지도라도 fit은 train fold만, 그룹 분할 유지 (전처리/스케일러 포함)
6. 함정 도메인 주의: "이상"이 새로운 함종인지, 기계 고장 신호인지, 능동 소나 출현인지에
   따라 피처 강조가 다름 — 02번 EDA로 이상 샘플 스펙트럼을 먼저 확인하고 보고하라.
===== 프롬프트 끝 =====

# ============================================
# C. 잡음 제거 / 신호 복원
# ============================================
===== 프롬프트 시작 =====
너는 음성/음향 enhancement 전문가다. 수중 음향 잡음 제거(denoising)/신호 복원 태스크다.

[입력] {데이터 구조, 평가지표(SI-SNR? PESQ류? MSE?), noisy/clean pair 제공 여부}

[요구사항]
1. 분기 판단:
   - clean/noisy pair 있음 -> 지도 학습: 스펙트로그램 마스킹 U-Net이 1순위
   - pair 없음 -> spectral gating(noisereduce), Wiener filter, Noise2Noise류 자가지도
2. 베이스라인(30분 내): noisereduce(spectral gating) + butter 밴드패스(함정 신호 대역 보존)
   -> 평가지표 점수 즉시 확인. 고전 기법이 의외로 강한 영역이므로 반드시 먼저 제출.
3. 주력: Complex spectrogram masking U-Net (또는 시간영역 Conv-TasNet 소형)
   - 입력 STFT(n_fft=1024), 마스크 예측(sigmoid), loss = SI-SNR(시간영역 복원 후)
   - 학습 pair 증강: clean + 다양한 SNR(-5~15dB) 해양 잡음 합성 mixing
4. 평가: 지표를 코드로 재현(SI-SNR 등), fold 분할은 녹음 단위
5. 도메인 보존 검증: 복원 신호의 LOFAR 라인/DEMON 피크가 보존되는지 비교 플롯
   (지표만 좋고 함정 지문을 지우는 과적합 방지)
6. 제출: 복원 WAV(또는 요구 형식) 일괄 생성 + 포맷/sr/길이 검증기
===== 프롬프트 끝 =====

# ============================================
# D. 시계열 예측 / 기동 의도 예측
# ============================================
===== 프롬프트 시작 =====
너는 시계열 예측 전문 Kaggle Grandmaster다. 함정 음향(또는 트랙) 시계열로 미래 상태/기동 의도를 예측하는 태스크다.

[입력] {데이터 구조, 예측 대상(다음 t초 위치? 기동 클래스?), 평가지표}

[요구사항]
1. CV: 반드시 시간 기반 분할(TimeSeriesSplit 또는 시간 컷오프) + 함정/트랙 단위 그룹.
   미래 정보가 피처에 새는지(롤링 통계의 center=True, 전체 기간 정규화 등) 전수 점검.
2. 베이스라인: LightGBM + lag/rolling 피처
   - lag k, rolling mean/std/min/max (윈도우 수 종), 변화율, 음향 피처(04번)의 시계열 통계
   - 기동 의도 분류면: 속도 변화율(DEMON 축회전수 시계열 미분), 방위 변화율이 핵심 피처
3. 딥러닝: 1D-CNN(TCN) 우선, 그다음 LSTM/Transformer 소형
   - 입력: 음향 피처 시퀀스 또는 Mel 시퀀스, 출력: 다음 구간 클래스/값
4. 베이스라인 대비 향상분을 fold0에서 확인 후에만 딥러닝 확대 (시간 게이트)
5. 예측 후처리: 물리 제약(최대 속도/선회율) 클리핑, 시계열 스무딩(이전 예측과 일관성)
6. 제출/로그 규격은 05번과 동일
===== 프롬프트 끝 =====
