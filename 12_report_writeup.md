# 12. 발표/보고서용 방법론 정리 프롬프트
목적: 마지막 3시간에 심사용 문서를 빠르게 생성. 기술 점수 외 발표 점수 확보.

===== 프롬프트 시작 =====

너는 ML 경진대회 솔루션 라이트업(write-up) 전문가다. 아래 실험 기록을 바탕으로 심사위원(해군 관계자 + AI 전문가 혼성)을 위한 방법론 보고서를 작성하라.

[입력]
- 실험 관리 시트: {시트 내용 붙여넣기}
- 최종 솔루션 구성: {모델/피처/앙상블 가중치}
- 최종 CV/public 점수: {점수}
- EDA 주요 발견: {02번 결론}

[보고서 구조 — 이 순서로]
1. 요약 (5줄): 태스크, 최종 접근, 점수, 핵심 차별점
2. 문제 정의 및 평가지표 분석: 지표 특성이 전략에 미친 영향
3. EDA 핵심 발견: 스펙트럼 분석 결과, 클래스 분리 정보가 어느 대역에 있었는지
   (LOFAR 라인/DEMON 변조 그림 1~2장 인용)
4. 검증 전략: 왜 녹음 단위 GroupKFold인가 — 세그먼트 랜덤 분할 대비 점수 차이 수치로
   누수 위험을 증명한 과정 (심사 신뢰도 핵심 포인트)
5. 솔루션 아키텍처: 파이프라인 다이어그램(텍스트), 모델별 역할, 앙상블 구성
6. 도메인 지식 활용: LOFAR/DEMON 피처가 점수에 기여한 근거
   (feature importance, ablation 수치)
7. 실험 여정: 시도 -> 결과 -> 판단 의 표 (실패 실험 포함 — 체계적 접근 어필)
8. 재현성: seed/체크포인트/inference.py 구조, 실행 방법
9. 한계와 실전 적용 방향: 실해역 도메인 갭, 저소음 함정 대응,
   실시간 처리 관점의 향후 과제 (해군 심사위원 대상 어필 포인트)

[톤]
- 비전문가도 따라올 수 있게 도메인 용어는 1줄 설명 병기
- 모든 주장에 수치 근거, 과장 금지
- 분량: A4 4~6페이지 분량의 마크다운

추가로 같은 내용을 발표 슬라이드 10장 구성안(장표별 제목 + 핵심 메시지 + 들어갈 그림)으로도 출력하라.

===== 프롬프트 끝 =====

## 미리 준비할 것 (대회 중 수집)
- [ ] EDA 그림 중 "한 장으로 설명되는" 클래스별 스펙트럼 비교 figure
- [ ] 랜덤 vs 그룹 CV 점수 비교 수치 (누수 증명 스토리)

- [ ] confusion matrix 최종본
