# 21. 다채널 하이드로폰 / 방위·거리 추정 태스크 프롬프트
목적: (a) 입력이 멀티채널 어레이인 경우, (b) 표적 방위(DOA)/거리 추정이 출제되는 경우 대비.

===== 프롬프트 시작 (A. 다채널 입력 처리) =====

너는 어레이 신호처리 + 딥러닝 전문가다. 다채널(C채널) 하이드로폰 수중 음향 데이터를 분류 파이프라인에 통합하는 코드를 작성하라.

[요구사항]
1. 채널 정찰: 채널 간 RMS/스펙트럼 차이, 채널 간 상관, 불량 채널(무신호/클리핑) 탐지
2. 처리 전략 4종 구현 + fold0 비교:
   a. 단순: 채널 평균(mono) — 베이스라인
   b. 채널별 독립 샘플 취급(데이터 N배) + 추론 시 채널 평균 — 증강 효과
   c. 멀티채널 입력: Mel을 채널축으로 쌓아 CNN 입력 (C,128,T)
   d. 공간 피처 추가: 채널쌍 GCC-PHAT(시간지연 상관) 벡터를 보조 입력으로 concat
3. 어레이 기하 정보가 제공되면: delay-and-sum 빔포밍으로 SNR 개선 채널 생성 옵션
4. 주의: 채널 간 미세 차이가 녹음 세션 지문이 될 수 있음 — 채널 통계가
   레이블과 상관되면 누수 경고 (03번 점검에 추가)

===== 프롬프트 끝 (A) =====

===== 프롬프트 시작 (B. 방위/거리 추정 회귀) =====

너는 수중 음향 측위 전문가다. 음향 신호로 표적의 방위각(DOA)/거리를 추정하는 태스크다.

[입력] {레이블 형식: 각도(0~360)? 거리(m)? 구간 클래스?}, {단일/다채널}, {평가지표}

[요구사항]
1. 각도 회귀의 순환성 처리 (필수):
   - 타깃을 (sin, cos) 2차원으로 변환해 회귀 -> atan2로 복원
   - 손실: MSE(sin,cos) 또는 1-cos(pred-true) 각도 손실
   - 평가: 순환 오차 mean(|wrap(pred-true)|) — 19번 metrics에 추가
2. 다채널이면: GCC-PHAT/위상차(IPD) 피처가 1순위 입력
   (방위 정보는 채널 간 시간지연에 있음 — Mel만으로는 부족함을 주석으로 명시)
   단일 채널이면: 거리 추정은 전달손실 단서(고주파 감쇠 기울기, 다중경로 패턴) 피처
3. 모델: LGBM(GCC/IPD 통계 피처) + CNN(위상 스펙트로그램 cos/sin 2채널 입력) 비교
4. 거리 추정: log 변환 회귀 + 물리 범위 클리핑, 지표가 상대오차면 SMAPE 직접 최적화
5. CV: 녹음/트랙 단위 그룹 분할 — 같은 트랙의 인접 시점이 train/valid에 갈리면 누수
6. 후처리: 시간 연속 트랙이면 칼만 필터/이동 중앙값 스무딩 (물리 일관성)

===== 프롬프트 끝 (B) =====

## 체크리스트
- [ ] 각도 타깃을 sin/cos로 변환했는가 (0/360 경계 오류는 치명적)
- [ ] 다채널 위상 정보를 버리지 않았는가
- [ ] 트랙 단위 그룹 분할인가
