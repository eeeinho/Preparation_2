# 01. 데이터 구조 점검 프롬프트
목적: 대회 시작 직후 10분 내 데이터 전체 구조 파악. 모델링 판단의 근거 확보.

===== 프롬프트 시작 =====

너는 Kaggle Grandmaster급 ML 엔지니어다. 해군 AI 경진대회(수중 음향 신호 분석, H200 GPU 원격 SSH, 인터넷 가능)가 방금 시작되었고, 모델링 전에 데이터 구조를 10분 안에 완전히 파악해야 한다.

[입력]
- 데이터 루트 경로: {DATA_DIR}
- 대회 안내문 요약: {대회 설명/평가지표/제출 형식 붙여넣기}

[요구사항]
아래 항목을 모두 수행하는 단일 파이썬 스크립트를 작성하라. placeholder 금지, 즉시 실행 가능해야 한다.

1. 폴더 트리 출력 (깊이 3, 디렉토리별 파일 수 요약)
2. 오디오 파일 전수 조사: 개수, 확장자, sample rate 분포, 채널 수 분포, 길이(초) 분포(min/mean/max/percentile), 총 시간
3. 깨진/읽기 불가 WAV 탐지 (try-except로 전수 스캔, 실패 파일 목록 저장)
4. 레이블 파일(csv) 구조: 컬럼, 클래스 분포(개수+비율), 결측, 멀티라벨 여부
5. train/test 파일명 패턴 분석: 정규식으로 공통 패턴 추출, train-test 파일명 중복 확인
6. 같은 원본 녹음에서 잘린 클립 추정:
   - 파일명 prefix/번호 패턴 그룹핑
   - 파일 길이가 정확히 동일한 그룹 탐지
   - (가능하면) 인접 클립 경계의 파형 연속성 검사
7. sample_submission 구조: 컬럼, 행 수, test 파일 수와 일치 여부, 제출 단위(파일? 세그먼트?) 판정
8. 클래스별 sample rate/길이 차이 검사 — 다르면 "메타데이터 누수 위험" 경고 출력
9. 마지막에 요약 리포트 출력:
   - 태스크 유형 추정(분류/탐지/회귀/잡음제거)
   - 권장 그룹 키(CV용)
   - 발견된 누수 위험 목록
   - 다음 단계 권고

[제약]
- soundfile.info 사용(전체 로드 금지, 속도 우선), 전수 스캔은 멀티프로세싱
- 결과는 print + recon_report.json 저장
- 코드 길이보다 누락 없는 점검이 우선

===== 프롬프트 끝 =====
