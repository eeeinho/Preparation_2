# 23. 대회 시작 직후 체크리스트 (0~15분, 사람용)
목적: 규정 오해/제출 형식 착오로 인한 실격·헛수고 방지. 보조1 팀원 전담.

## A. 규정 파악 질문 (안내문에서 답을 찾아 팀 채팅에 고정)
- [ ] 평가지표 정확한 정의 (macro? weighted? 녹음 단위? 세그먼트 단위?)
- [ ] 제출 횟수 제한 / public-private 분할 여부와 비율
- [ ] 제출물 범위: csv만? 코드+가중치? 추론 재현 요구? 보고서/발표 포함?
- [ ] 외부 데이터 사용 허용 여부 (DeepShip/ShipsEar 사전학습 가능한가 — 허용 시 큰 무기)
- [ ] 사전학습 모델 허용 여부 (AudioSet/음성 사전학습 — 대부분 허용이나 명시 확인)
- [ ] 생성형 AI/외부 API 사용 규정 (18번 발동 조건)
- [ ] 인터넷 사용 범위 (검색만? 다운로드? API?)
- [ ] 팀 간 공유 금지 조항, 코드 표절 기준
- [ ] 마감 시각과 제출 방식 (서버 업로드? 별도 플랫폼? 시간대 확인)
- [ ] 심사 배점: 점수 외 항목(발표/창의성/생성형AI 활용)이 있으면 시간 배분 조정

## B. 즉시 결정 사항 (InhoLee)
- [ ] 평가지표 -> 19번 metrics.py에서 매칭, 전 파이프라인 기준 통일 선언
- [ ] 태스크 유형 -> 사용할 프롬프트 경로 선택:
      분류(기본 세트) / 이상탐지·잡음제거·시계열(10번) / SED(20번) / 다채널·DOA(21번)
- [ ] 외부 데이터 허용 시: DeepShip/ShipsEar 다운로드를 백그라운드 시작
      (사전학습 -> 본선 fine-tune 2단계 전략, 08번에 통합)
- [ ] 점수 외 배점 존재 시: 12번 보고서 + 18번 문서화를 초반부터 병행 가동

## D. 외부 데이터 활용 전략 (허용 시 — 사전 숙지)
- DeepShip(4함종, 47h)/ShipsEar(5그룹+배경): 본선 클래스와 다르더라도
  백본 사전학습(함종 분류 과제로 선학습) -> 본선 데이터 fine-tune이 정석
- 본선 데이터가 소규모일수록 효과 큼. 단, 도메인 갭(해역/장비) 존재 —
  사전학습 유/무 fold0 비교 후 채택
- 배경 잡음 소스로도 활용 가능 (15번 합성 mixing 재료)
