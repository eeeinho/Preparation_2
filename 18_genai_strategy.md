# 18. 생성형 AI 활용 전략 프롬프트
목적: 대회명에 "생성형 AI 활용"이 들어가는 만큼, 심사 평가 항목일 가능성에 대비.
실점수 기여 + 활용 과정 문서화(심사 어필) 양쪽 대응.

===== 프롬프트 시작 =====

너는 생성형 AI 엔지니어다. 수중 음향 분류 대회에서 생성형 AI(Claude/GPT)를 점수와 심사 평가 양면에서 활용하는 전략을 실행 코드와 함께 제시하라.

[활용 카테고리 — 각각 구현/문서화]
1. 개발 가속 (기본):
   - 역할 분리 세션 운영: 코드 생성용 / 디버깅용 / 전략 검토용(악마의 변호인 역할
     부여 — "이 CV 설계의 허점을 찾아라") 세션을 분리해 컨텍스트 오염 방지
   - 모든 프롬프트와 결과를 prompts_log.md에 기록 (심사 제출용 증빙)
2. LLM 보조 라벨링/검수 (조건부):
   - 오분류 상위 샘플의 스펙트로그램 이미지를 멀티모달 LLM에 입력해
     "LOFAR 라인 개수/위치, DEMON 변조 유무" 구조화 응답(JSON) 수집
   - 사람 청취 검수와 교차 확인 -> 라벨 노이즈 후보 식별
3. LLM 피처(실험적, 시간 여유 시만):
   - 스펙트로그램 이미지 -> 멀티모달 LLM -> 구조화 속성(추정 날수, 라인 수,
     광대역/협대역 우세 등) -> LGBM 추가 피처
   - 반드시 OOF로 기여도 검증, API 비용/시간 대비 효과 없으면 즉시 중단
   - 주의: test 피처 생성 시 동일 프롬프트/온도 0 고정 (재현성)
4. 합성 데이터 보조 (15번 연계):
   - LLM에게 잡음 합성 파라미터 공간(SNR/잡음 유형 조합) 설계와
     ablation 실험 매트릭스 생성을 위임
5. 보고서/발표 생성 (12번 연계) + 프롬프트 엔지니어링 기법 명시:
   - CoT, self-consistency(전략 결정 시 3회 독립 질의 후 다수결),
     역할 프롬프트 — 사용한 기법을 보고서 "생성형 AI 활용" 섹션에 정리

[요구사항]
1. Anthropic API 호출 유틸 (배치, 재시도, JSON 스키마 강제, 캐싱) 코드
2. 스펙트로그램 -> 이미지 -> API 멀티모달 파이프라인 코드 (2,3번용)
3. prompts_log.md 자동 기록 데코레이터
4. 비용/시간 가드: 호출 수 상한, 누적 비용 추정 출력
5. 보고서용 "생성형 AI 활용 요약" 섹션 자동 생성 함수
   (카테고리별 활용 내용, 기여 수치, 사용 기법)

[원칙]
- LLM 출력은 항상 검증 가능한 형태(JSON)로 받고 OOF로 효과 검증
- 점수 기여 없는 활용은 보고서에 "시도와 한계"로 솔직히 기록 (신뢰도 어필)
